## API Endpoints to Build
1. GET  /health      → { status: ok, vector_count: int }
2. POST /ingest      → Load JSON from /data, chunk, embed, store in ChromaDB
3. POST /query       → Accept question, retrieve context, call LLM, return answer
## /query Request + Response Schema
Request:  { question: str, match_id: str | None, top_k: int = 5 }
Response: { answer: str, sources: list[str], confidence: float,
           match_id: str, processing_time_ms: int }
## RAG Pipeline Logic (rag.py)
1. Embed the question using OpenAI text-embedding-3-small
2. Retrieve top_k most similar chunks from ChromaDB
3. Filter chunks by match_id if provided
4. Build prompt: system_prompt + retrieved_context + user_question
5. Call Groq API (llama-3.1-8b-instant)
6. Return answer + source chunk IDs + processing time
## System Prompt for LLM (use exactly this)
SYSTEM_PROMPT = """
You are FootIQ, an AI assistant for football coaches.
Answer ONLY using the match data provided in the context below.
If the answer is not in the context, say:
  'I don't have enough match data to answer this.'
Never guess or fabricate statistics.
Keep answers concise, factual, and coach-friendly.
"""
## StatsBomb Data Ingestion Logic (loader.py)
- Load all JSON files from /data directory
- Each JSON = one match event stream (list of event dicts)
- Convert each event to natural language string:
  e.g. "Minute 23: Messi (Barcelona) completed a pass to Xavi"
- Chunk into groups of 20 events per document chunk
- Store metadata: { match_id, chunk_index, minute_start, minute_end }
- Embed and store in ChromaDB collection: 'footiq_matches'
## Error Handling Rules
- Empty question → 422 Unprocessable Entity
- No match data in ChromaDB → 503 with clear message
- Groq API failure → retry once, then 502 Bad Gateway
- match_id not found → return global search across all matches
- Always log: question, match_id, chunk_count, latency_ms

2.5 Confidence Score Logic
Confidence is computed from ChromaDB similarity scores — not from the LLM itself:
def compute_confidence(similarity_scores: list[float]) -> float:
    # ChromaDB returns L2 distance (lower = more similar)
    # Convert to 0-1 confidence: 1 - normalized_distance
    if not similarity_scores:
        return 0.0
    avg_distance = sum(similarity_scores) / len(similarity_scores)
    # Clamp to reasonable range (0 to 2 for L2 distance)
    confidence = max(0.0, min(1.0, 1 - (avg_distance / 2)))
    return round(confidence, 3)
