1. Product Overview
1.1 Problem Statement
Football coaches at club and academy levels spend hours manually reviewing 90-minute match footage and fragmented stat sheets. This results in delayed tactical decisions, inconsistent player development tracking, and missed opportunities to act on real-time performance data.
There is no intelligent, conversational tool that lets coaches simply ask questions about a match and get instant, grounded answers backed by real match data.
1.2 Product Vision
"
FootIQ is the AI copilot that turns every match into a coaching advantage — giving football coaches instant, intelligent answers about player performance, tactics, and game patterns through natural language.

3.2 Feature Deep-Dive — F01: Match Q&A
User Story
As a club coach, I want to ask plain-English questions about a match (e.g. 'How many times did we lose possession in the final third?') and get an instant, data-backed answer — so I can make tactical decisions without reviewing hours of footage.
Acceptance Criteria
Coach can type a free-text question about any uploaded match
System retrieves relevant match data chunks via vector search (ChromaDB)
LLM generates a response grounded strictly in retrieved data (no hallucination)
Response includes the specific stat or event that answers the question
Response delivered in < 5 seconds for 95% of queries
If data is insufficient, system says so — never fabricates
3.3 Feature Deep-Dive — F02: Player Performance Summary
User Story
As an academy trainer, I want an auto-generated performance card for each player after every session — so I can track development without spending 2 hours on spreadsheets.
Acceptance Criteria
System generates a per-player summary card on match upload completion
Card includes: distance covered, pass accuracy, duels won/lost, key contributions
AI narrative paragraph (2-3 sentences) summarising player's overall performance
Cards are exportable as PDF per player
Generation completes within 30 seconds of match upload

7. Appendix
7.1 Glossary
Term
Definition
RAG
Retrieval-Augmented Generation — grounding LLM answers in retrieved documents
ChromaDB
Open-source local vector database for storing and searching embeddings
LangChain
Python framework for building LLM-powered applications and RAG pipelines
YOLOv8
State-of-the-art real-time object detection model used for player detection
ByteTrack
Multi-object tracking algorithm for tracking detected players across video frames
StatsBomb
Football data company providing free open-data match event datasets
Embedding
Numerical vector representation of text used for semantic similarity search
Hallucination
When an LLM generates plausible-sounding but factually incorrect information

Live match real-time analysis (requires streaming infrastructure)
